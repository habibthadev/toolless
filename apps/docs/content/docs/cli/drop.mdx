---
title: toolless drop
description: Drop databases or collections
---

The `drop` command permanently removes databases or collections and all their data.

## Usage

```bash title="Terminal"
toollessdb drop <database> [collection] [options]
```

## Options

## Examples

### Drop Entire Database

```bash title="Terminal"
toollessdb drop myapp
```

This prompts for confirmation before deleting the entire `myapp` database.

```
⚠ This will permanently delete database "myapp" and all its collections.
Are you sure? (y/N):
```

### Drop Specific Collection

```bash title="Terminal"
toollessdb drop myapp users
```

Prompts before deleting only the `users` collection:

```
⚠ This will permanently delete collection "users" from database "myapp".
Are you sure? (y/N):
```

### Force Drop (No Confirmation)

```bash title="Terminal"
toollessdb drop myapp --force
```

<Callout type="warn">
  The `--force` flag bypasses confirmation. Use with extreme caution in scripts.
</Callout>

### Drop from Custom Path

```bash title="Terminal"
toollessdb drop myapp users -p /path/to/databases
```

## Safety

- Requires explicit confirmation unless `--force` is used
- Shows what will be deleted before proceeding
- Cannot be undone—data is permanently removed
- Database files are deleted from the filesystem

<Callout type="error">
  **Warning**: Dropped data cannot be recovered. Always backup important data before using this
  command.
</Callout>

## Alternative: Backup Before Drop

```bash title="Terminal"
toollessdb export myapp users backup.jsonl
toollessdb drop myapp users
```

This exports the collection before dropping it, allowing recovery if needed.
